 Bazadan yozuvlar tanlash (Select). AND, OR va NOT operatorlari;
MySQL‚Äôda yozuvlarni tanlash (SELECT) ‚Äî bu SQL so'rovi orqali ma'lumotlar bazasidagi ma'lumotlarni tanlash va ko'rsatish jarayonidir. Bu jarayonda AND, OR, va NOT operatorlari yordamida ma'lumotlarni yanada aniqroq va shartlarga mos ravishda tanlash mumkin.

SELECT ism, yosh
FROM talabalar

Bu so‚Äòrovda:
yosh > 20 va ism = 'Ali' shartlari AND operatori bilan birlashtirilgan.

FROM talabalar

yosh > 20 yoki ism = 'Ali' shartlaridan bittasi true bo‚Äòlsa, natija qaytariladi.
Demak, yosh 20 dan katta bo‚Äòlgan talabalar yoki ismi "Ali" bo‚Äòlgan talabalar ko‚Äòrsatiladi.

Bu operatorlar SELECT, UPDATE, DELETE kabi SQL so‚Äòrovlarida shartlarni aniqlash uchun ishlatiladi.
Taqqoslash operatorlari
Taqqoslash operatorlari ma'lumotlar ustida ikki yoki undan ortiq qiymatni solishtirish uchun qo‚Äòllaniladi. Bu operatorlar yordamida foydalanuvchilar ma'lum qiymatlarga teng, katta, kichik yoki boshqa shartlarga javob beruvchi yozuvlarni tanlab olishlari mumkin.
MySQL'da qo‚Äòllaniladigan asosiy taqqoslash operatorlari quyidagilardan iborat:
= ‚Äî Tenglik operatori. Misol: age = 25
!= yoki <> ‚Äî Teng emaslik operatori. Misol: salary != 1000
> ‚Äî Kattaroqlik operatori. Misol: price > 500
< ‚Äî Kichikroqlik operatori. Misol: price < 1000
>= ‚Äî Kattaroq yoki teng. Misol: rating >= 4.5
<= ‚Äî Kichikroq yoki teng. Misol: discount <= 10
BETWEEN ... AND ... ‚Äî Belgilangan oraliqda. Misol: age BETWEEN 18 AND 30
IN (...) ‚Äî Berilgan qiymatlar ro‚Äòyxatida mavjudligini tekshiradi. Misol: city IN ('Toshkent', 'Samarqand')
LIKE ‚Äî Matnli andozalarga mos kelishini aniqlaydi. Misol: name LIKE 'A%'
IS NULL ‚Äî NULL qiymat mavjudligini tekshiradi. Misol: email IS NULL
IS NOT NULL ‚Äî NULL qiymat mavjud emasligini bildiradi. Misol: email IS NOT NULL
Mantiqiy operatorlar
Mantiqiy operatorlar bir nechta shartlarni birgalikda qo‚Äòllash va ularning mantiqiy bog‚Äòliqligini aniqlash uchun xizmat qiladi. Ular yordamida murakkab shartli ifodalarni yaratish mumkin.

52. MySQL da Viewlar
MySQL‚Äôda View (qarash) ‚Äì bu soxta (virtual) jadvallardir. Ular real ma‚Äôlumotlarni o‚Äòzida saqlamaydi, balki asosiy jadvallardagi ma‚Äôlumotlarga asoslangan holda so‚Äòrov (query) natijalariga tayangan holda ishlaydi. View‚Äôlar orqali murakkab so‚Äòrovlarni soddalashtirish, xavfsizlikni oshirish va ma‚Äôlumotlar bazasi tuzilmasini tartibga solish mumkin. View bu ‚Äì xuddi oddiy jadval kabi ishlatiladi, lekin u aslida SELECT operatoriga asoslangan saqlangan so‚Äòrov hisoblanadi. View yaratilganda unda hech qanday real ma‚Äôlumot saqlanmaydi ‚Äì u faqatgina kerakli so‚Äòrovni har safar bajaradi va natijasini qaytaradi.
View yaratishning asosiy sabablari quyidagilardan iborat: birinchidan, murakkab SQL so‚Äòrovlarini qayta-qayta yozmaslik, ularni oddiy nom orqali chaqirish imkoniyatidir; ikkinchidan, xavfsizlik ‚Äì foydalanuvchiga faqat kerakli ustunlarni yoki yozuvlarni ko‚Äòrsatish orqali ma‚Äôlumotlarning to‚Äòliq emas, balki cheklangan qismi taqdim etiladi; uchinchidan, loyihaning strukturaviy soddaligi ‚Äì ko‚Äòplab jadvallar o‚Äòrniga yagona ko‚Äòrinish taqdim etiladi.
View quyidagi sintaksis orqali yaratiladi:
CREATE VIEW view_nomi AS SELECT_izohi;
Misol uchun, agar bizda students jadvali mavjud bo‚Äòlib, unda barcha talabalarning shaxsiy ma‚Äôlumotlari saqlansa, faqat ismi, familiyasi va guruhini o‚Äòz ichiga olgan View quyidagicha yaratiladi:
CREATE VIEW student_short_info AS 
SELECT name, surname, group_name 
FROM students;
View‚Äôlar faqat o‚Äòqish uchun emas, ba‚Äôzi hollarda yangilash, o‚Äòchirish yoki qo‚Äòshish uchun ham ishlatilishi mumkin. Biroq, bu faqatgina oddiy View‚Äôlar uchun amal qiladi. Agar View murakkab bo‚Äòlsa (ya‚Äôni, JOIN, GROUP BY, DISTINCT, AGGREGATE FUNCTION kabi operatorlar ishlatilgan bo‚Äòlsa), unda yozish (INSERT, UPDATE, DELETE) operatsiyalari cheklangan bo‚Äòladi.
Xulosa qilib aytganda, View‚Äôlar MySQL ma‚Äôlumotlar bazasida ma‚Äôlumotlarga ishlov berish, ularni soddalashtirish, xavfsizlikni ta‚Äôminlash va arxitektura yengilligini yaratishda muhim vosita hisoblanadi. View‚Äôlar orqali murakkab so‚Äòrovlar oddiy ko‚Äòrinishda taqdim etiladi va bu esa dasturchilar hamda oxirgi foydalanuvchilar uchun qulaylik yaratadi. Shu bilan birga, View‚Äôlar real ma‚Äôlumot saqlamagani sababli ularning ishlashi tez va samarali bo‚Äòladi.


53. MySQL‚Äôda saqlanadigan proseduralar
MySQL‚Äôda saqlanadigan proseduralar (stored procedures) ‚Äì bu ma‚Äôlumotlar bazasida bir marta yaratiladigan va keyinchalik takror-takror chaqiriladigan SQL buyruqlar majmuasidir. Ular odatda murakkab yoki takrorlanuvchi amallarni avtomatlashtirish, kodni modullashtirish va ma‚Äôlumotlar bazasi ishlashini optimallashtirish maqsadida qo‚Äòllaniladi. Saqlanadigan proseduralar yordamida foydalanuvchi bir necha SQL operatorlarini yagona blokda jamlab, ularni parametrlar bilan boshqarishi mumkin. Bunday yondashuv dastur kodini soddalashtiradi, xatoliklarni kamaytiradi va ishlash samaradorligini oshiradi. Prosedura server darajasida saqlanadi va chaqirilganda avtomatik ravishda bajariladi.

DELIMITER //

54. MySQL‚Äôda tranzaksiyalar
MySQL‚Äôda tranzaksiya (transaction) ‚Äì bu ma‚Äôlumotlar bazasida bajarilishi kerak bo‚Äòlgan bir yoki bir nechta SQL amallarining mantiqiy guruhidir. Tranzaksiyalar yordamida ma‚Äôlumotlar ustida bir nechta amallarni ketma-ket bajarish va ularning yagona operatsiya sifatida amalga oshirilishini ta‚Äôminlash mumkin. Tranzaksiya to‚Äòliq bajarilishi yoki umuman bajarilmasligi kerak. Bu yondashuv ACID tamoyillariga asoslanadi: Atomicity (atomiklik), Consistency (izchillik), Isolation (izolyatsiya), Durability (barqarorlik). MySQL‚Äôda tranzaksiyalar asosan InnoDB turidagi jadvallar bilan qo‚Äòllab-quvvatlanadi.
Atomiklik ‚Äì bu tranzaksiyadagi barcha operatsiyalar yoki to‚Äòliq bajariladi, yoki umuman bajarilmaydi. Agar biror bosqichda xatolik yuz bersa, butun tranzaksiya bekor qilinadi. Izchillik ‚Äì tranzaksiya bajarilishidan oldin va keyin ma‚Äôlumotlar bazasi izchil holatda bo‚Äòlishi kerak. Izolyatsiya ‚Äì har bir tranzaksiya mustaqil bajariladi va boshqa tranzaksiyalarga ta‚Äôsir qilmaydi. Barqarorlik ‚Äì tranzaksiya bajarilib, natijalar saqlangandan so‚Äòng ular tizim nosozligiga qaramay saqlanib qoladi.
MySQL‚Äôda tranzaksiya quyidagi buyruqlar orqali boshqariladi:
START TRANSACTION yoki BEGIN ‚Äì tranzaksiyani boshlash
COMMIT ‚Äì tranzaksiyani yakunlash va o‚Äòzgarishlarni saqlash
ROLLBACK ‚Äì xatolik yuz bersa tranzaksiyani bekor qilish va ma‚Äôlumotlarni dastlabki holatga qaytarish
Quyidagi misolda tranzaksiya orqali bank hisoblaridan pul o‚Äòtkazish jarayoni ifodalangan:
START TRANSACTION;
UPDATE accounts SET balance = balance - 500 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 500 WHERE account_id = 2;
COMMIT;
Tranzaksiyalar ma‚Äôlumotlar bazasining ishonchliligini ta‚Äôminlashda, ayniqsa ko‚Äòp foydalanuvchili va real vaqtli tizimlarda muhim rol o‚Äòynaydi. Ular yordamida ma‚Äôlumotlar ustida bajariladigan amallar to‚Äòliq nazorat ostida bo‚Äòlib, noto‚Äòg‚Äòri yoki to‚Äòliq bajarilmagan amallar tufayli yuzaga keladigan xatoliklar oldi olinadi.

            }
        }

SQL So‚Äòrovi:
Oxirgi qo‚Äòshilgan talabaning ID sini olish uchun SQL so‚Äòrovi quyidagicha bo‚Äòladi:
SELECT LAST_INSERT_ID();
Bu so‚Äòrov oxirgi qo‚Äòshilgan yozuvning auto-increment maydonini (masalan, student_id) qaytaradi.

C# da oxirgi qo‚Äòshilgan talabaning ID sini olish uchun quyidagi kodni ishlatish mumkin:
using MySql.Data.MySqlClient;
using System;
public class MySQLExample
{
    public static void Main(string[] args)
    {
        // MySQL serveriga bog‚Äòlanish uchun connection string
        string connectionString = "Server=localhost;Port=3306;Database=mydatabase;Uid=myusername;Pwd=mypassword;";
        try
        {
            // Bog‚Äòlanishni yaratish
            using (MySqlConnection conn = new MySqlConnection(connectionString))
            {
                // Bog‚Äòlanishni ochish
                conn.Open();
                Console.WriteLine("MySQL serverga bog‚Äòlandik!");
                // Yangi talabani qo‚Äòshish (INSERT)
                string insertQuery = "INSERT INTO students (name, age) VALUES ('Ali', 22)";
                MySqlCommand cmdInsert = new MySqlCommand(insertQuery, conn);
                cmdInsert.ExecuteNonQuery();  // INSERT so‚Äòrovini bajarish

                }
                else
                {
                    Console.WriteLine("ID olishda xatolik yuz berdi.");
                }
            }
        }
        catch (Exception ex)
        {
            // Xatolikni ushlash
            Console.WriteLine("Xato: " + ex.Message);
        }
    }
